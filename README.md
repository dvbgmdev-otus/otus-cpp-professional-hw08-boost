# bayan

`bayan` - утилита для поиска файлов-дубликатов по содержимому.

Исходная постановка задания сохранена в [TASK.md](TASK.md).

## Назначение

Утилита сканирует указанные директории, отбирает файлы по размеру и маскам имён, затем сравнивает кандидатов блоками фиксированного размера. Каждый блок хэшируется выбранным алгоритмом. Файлы считаются дубликатами, если совпадает вся последовательность хэшей блоков.

Чтение файлов выполняется лениво: блок читается только когда он нужен для сравнения, а рассчитанный хэш блока кэшируется.

## Использование

```bash
bayan [options]
```

Пример:

```bash
bayan \
  --include ./docs ./src \
  --exclude ./build \
  --level 2 \
  --min-size 2 \
  --mask "*.txt" "*.md" \
  --block-size 4096 \
  --hash crc32
```

## Параметры

| Параметр | Описание | Значение по умолчанию |
| --- | --- | --- |
| `--include DIR` | Директория для сканирования. Можно указать несколько директорий. | `.` |
| `--exclude DIR` | Директория для исключения из сканирования. Можно указать несколько директорий. | пусто |
| `--level N` | Глубина сканирования. `0` - только указанная директория без вложенных. | `0` |
| `--min-size N` | Минимальный размер файла в байтах. | `1` |
| `--mask MASK` | Маска имени файла. Можно указать несколько масок. Регистр не учитывается. | пусто, подходят все имена |
| `--block-size N` | Размер блока чтения в байтах. | `4096` |
| `--hash crc32\|md5` | Алгоритм хэширования блоков. | `crc32` |
| `--help` | Показать справку. | - |

## Маски файлов

Маски применяются к имени файла, а не к полному пути.

Поддерживаются wildcard-символы:

- `*` - любая последовательность символов;
- `?` - один произвольный символ.

Сравнение выполняется без учёта регистра.

Примеры:

```bash
--mask "*.txt"
--mask "*.cpp" "*.h"
```

## Формат вывода

Результат выводится в `stdout`.

- один файл на строку;
- файлы с идентичным содержимым идут подряд одной группой;
- разные группы разделяются пустой строкой;
- если дубликаты не найдены, вывод пустой.

Пример:

```text
/tmp/data/a.txt
/tmp/data/b.txt

/tmp/data/report-1.md
/tmp/data/report-copy.md
```

## Коды возврата

| Код | Значение |
| --- | --- |
| `0` | Успешное выполнение. |
| `1` | Ошибка параметров или выполнения. Текст ошибки выводится в `stderr`. |

## Сборка и проверка

```bash
./prj build
./prj test
```

Для очистки build-директории:

```bash
./prj clean
```

## Самопроверка сценария задания

Скрипт [tests/unit/self_check.sh](tests/unit/self_check.sh) запускает собранный бинарный файл `bayan` и проверяет сценарий, составленный по требованиям задания:

- несколько директорий сканирования;
- исключённая директория;
- уровень сканирования;
- минимальный размер файла;
- маска имени файла без учёта регистра;
- размер блока;
- алгоритм хэширования;
- группа из более чем двух дубликатов;
- отсутствие ложного срабатывания для файла того же размера, но с другим содержимым.

Скрипт запускается автоматически в составе:

```bash
./prj test
```

Его можно запустить отдельно после сборки:

```bash
./tests/unit/self_check.sh
```
